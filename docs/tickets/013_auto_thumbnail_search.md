# 013 書籍サムネイル自動取得機能の実装

## 1. 概要

### 目的

ユーザーが書籍メモを作成する際、タイトルと著者名から「実在する本の表紙画像（サムネイル）」を検索・選択できるようにし、写真撮影やアップロードの手間を削減する。

### 背景

- 読書メモ作成時の「画像を用意する」という心理的ハードルを下げ、継続利用を促す。
- 002（ユーザー認証）および 001（CustomUser）完了後のタスク。

---

## 2. 実装方針

### 基本方針

- **画像取得の自動化:** Google Books API を利用。
- **ユーザー選択式:** APIから取得した複数の候補（最大5件）からユーザーが選択するUIを採用。
- **データの整合性:** APIから取得したタイトル/著者名でユーザーの既存入力を強制上書きせず、ユーザーの入力を尊重する。
- **ストレージ戦略:** 画像の実ファイルは保存せず、APIから提供されるURLをデータベースに保持する（パターンA）。

---

## 3. データ構造の変更 (Models)

`Page` モデルに、取得したサムネイルURLを保持するフィールドを追加する。

```python
# wisme/models.py
class Page(models.Model):
    # ... 既存フィールド ...
    image_url = models.URLField(max_length=500, null=True, blank=True)
```

※ フィールド追加後はマイグレーションが必要です。既存データに対しては `null=True` で対応します。

---

## 4. 実装フローとロジック

### 4-1. ユーザー操作の流れ

1. **初期入力:** ユーザーが本の「タイトル」を入力。
2. **検索トリガー:** フォーム内の「画像検索で探す」ボタンをクリック。
3. **追加情報の入力:** 著者名入力欄（任意）を表示し、必要に応じて入力を促す。
4. **APIリクエスト:** 「検索実行」により Google Books API を呼び出す。
5. **候補表示:** ヒットした上位5件の「サムネイル画像 + タイトル」をリスト表示。
6. **選択・反映:** ユーザーが候補を選択すると、その `thumbnail_url` が隠しフィールド（`image_url`）にセットされ、プレビューが表示される。

### 4-2. API検索ロジック (Back-end / Utils)

検索クエリの精度を高めるため、以下の形式でAPIをリクエストする。

- **Endpoint:** `https://www.googleapis.com/books/v1/volumes`
- **Query Params:**
  - `q`: `intitle:{title}+inauthor:{author}` (著者名が空なら `intitle:{title}`)
  - `maxResults`: `5`
  - `printType`: `books`
  - `langRestrict`: `ja`
  - `key`: `settings.GOOGLE_BOOKS_API_KEY`

---

## 5. UI/UX 仕様

- **非同期通信:** ページのリロードを防ぐため、JavaScript (Fetch API) を使用して非同期でAPI結果を取得・表示する。
- **フォールバック:**
  - 検索結果が0件の場合、「見つかりませんでした」というメッセージを表示。
  - APIが利用できない場合に備え、手動での画像アップロード機能も予備として維持する。
- **Mixed Content対策:** APIから返されるURLが `http://` の場合、フロントエンドまたはサーバーサイドで `https://` に置換して扱う。
- **エラーハンドリング:** 画像のリンク切れ（`onerror`）に備え、フロントエンドでデフォルトの「No Image」画像を表示するロジックを組み込む。

## 6. テスト項目 (DoD: Definition of Done)

### 動作確認

- [ ] タイトル（＋著者名）からGoogle Books APIの検索結果が正しく取得できる。
- [ ] 5件の候補から1つを選択した際、正しいURLがフォームに反映され、保存される。
- [ ] 著者名が未入力の状態でも、タイトルのみで検索が動作する。
- [ ] 保存された `image_url` をもとに、ダッシュボードや詳細画面で表紙画像が表示される。

### 安全性・品質確認

- [ ] Google Books API キーが `.env` で管理され、公開リポジトリに含まれない設定になっている。
- [ ] 画像読み込みエラー時に、プレースホルダー画像が表示される。
- [ ] API制限（1,000回/日）を考慮し、不要なリクエストが発生しないUI設計になっている。
